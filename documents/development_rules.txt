Development Rules
============================
1)	Module, Class, Method, Attribute ต้องมีความหมายที่ชัดเจน และ Make Sense
2)	ใช้ Template ของบริษัทสำหรับส่วนที่ Implement เอง
3)	ต้องใส่ Comment ที่ Code ให้เพียงพอ เนื่องจากเราไม่มีเอกสาร Design อย่างละเอียด คนอื่นจะได้เข้าใจ Code ที่เราเขียน
4)	ก่อน Commit ลง Git จะต้อง Update Code จาก Git ให้เป็น Version ล่าสุดทุกครั้ง และทดสอบให้เรียบร้อยจนไม่สามารถหา Bug ได้ แล้วค่อย Commit
